为什么
把请求想成一根从左到右的链条：tools → system → CLAUDE.md/skills → messages。Anthropic的cache系统按这个顺序逐段算hash。改了链条中间某一段，从这段往右所有cache跟着失效，左边的保住。所以越靠左的段，改一次代价越大。

为什么
装新 MCP 动到 tools 层（链条最左边），hash 一变，下面 system 和 messages 跟着失效——三层一起重建。好在 MCP 只在 Claude Code 启动时读一次：session 跑着装新 MCP，当前 session 不受影响。

怎么办：启动前一次装好。装完别 /resume 也别 /reload-plugins——这两个命令会让 Claude Code 重读配置，tools 数组重新组装，cache prefix 对不上，之前的 cache 全丢。开新 session 而不是 /resume。

为什么
官方默认TTL是5分钟，超过这个时间没新请求，cache条目直接被删——不是hash对不上，是服务器主动清掉。回来再发消息，整段从头重新写入。

怎么办：启动 Claude Code 之前先 export ENABLE_PROMPT_CACHING_1H=1，cache 闲置 1 小时才过期。写入略贵，长会话命中一次就回本。
